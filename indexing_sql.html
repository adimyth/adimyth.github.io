Indexes in SQL

Link 1

  • Speeds up retrieval of rows by using a pointer.
  • Speeds up SELECT queries and WHERE clauses, but slows down data input that use UPDATE and INSERT statement.
  • The mysql indexes are stored as B-tree where all the data are in leaf node.
  • In MySQL you can use EXPLAIN in front of your SELECT statement to see if your query will make use of any index. This is a good start for troubleshooting performance problems.
  • If you have a three-column index on (col1, col2, col3), you have indexed search capabilities on (col1), (col1, col2), and (col1, col2, col3).
  • Comparison of dissimilar columns may prevent use of indexes if values cannot be compared directly without conversion. Example: 1 might equal to '0001', '1' or '01.e1'

Creating Indexes


            CREATE INDEX <index-name> ON TABLE <column-name>;

Showing Indexes


            SHOW INDEXES FROM <table-name>;

Dropping Indexes


            DROP INDEX <index-name>

When to create indexes

  • A column has wide range of unique values.
  • Multiple columns are used in a where or join clause.

Types of Indexes

Clustered Indexes

  • A clustered index determines the physical order of data in a table. Hence, only one clustered index can exist in a table.
  • The column on which the index is created is sorted according to some logical order.
  • In SQL Server, the primary key constraint automatically creates a clustered index on that particular column.
  • Example: Id is numerically sorted, name is alphabetically ordered.

Non-Clustered Indexes

  • A non-clustered index is stored at one place and table data is stored in another place. Similar to a textbook where the book content is located in one place and the index is located in another
  • Multiple non-clustered indexes are possible.
  • The data inside is stored as specified unlike clustered index.
  • When a query is issued against a column on which the index is created, the database will first go to the index and look for the address of the corresponding row in the table. It will then go to that row address and fetch other column values.
  • Slower than clustered index.

Unique Indexes

  • A unique index ensures that the values in the index key columns are unique even though the values in the columns are non-unique.

Full-Text Indexes

  • Clustered & Non-clustered indexes can be created on most columns except those columns configured with large object data types such as text, varchar.
  • Full-text indexes allows querying against character-based data.
  • Only one full-text index is allowed per table or view.
